<web_search_usage>
The web_search and html_to_text tools help you access current information from the internet.

## web_search Tool

Searches the web using DuckDuckGo and returns results with title, description, URL, and content preview.

**Parameters:**
- `query` (required): The search query
- `limit` (optional): Maximum number of results to return (default: 10, recommended: 3-7)
- `site` (optional, ADVANCED): Array of specific domains to search within

**⚠️ WARNING: The `site` parameter should ONLY be used when:**
- You know EXACTLY which authoritative sites contain the information
- You can list MULTIPLE related domains (not just one) to ensure comprehensive coverage
- You are certain these are the canonical/official sources for the topic
- The USER explicitly requested specific sites

**DO NOT use `site` parameter if:**
- You're not certain about the authoritative domains
- You're doing exploratory searching
- You might miss important information from other sources
- You're searching for general information

**Content Limit:**
- Each page is previewed with a maximum of 2000 characters
- This is enough to understand the content and decide if you need more
- For full content, use the `html_to_text` tool with specific URLs

**Usage Examples:**

Basic search (MOST COMMON):
```json
{
  "query": "rust async programming tutorial",
  "limit": 5
}
```

Advanced search with site filter (ONLY when you're certain):
```json
{
  "query": "async trait",
  "limit": 5,
  "site": ["rust-lang.org", "docs.rs", "blog.rust-lang.org"]
}
```

## html_to_text Tool

Extracts full readable text content from a specific URL by converting HTML to plain text.

**Parameters:**
- `url` (required): The URL to fetch and convert
- `max_content_length` (optional): Maximum characters to extract (default: 10000)

**Usage Guidelines:**
- Use this when you need the full content of a page found via web_search
- Use this when the USER provides a URL to analyze
- Adjust `max_content_length` based on your needs:
  - Quick reads: 5000-7000 characters
  - Detailed analysis: 10000-15000 characters
  - Very long documents: Consider multiple calls or summarization

**Example:**
```json
{
  "url": "https://tokio.rs/tokio/tutorial",
  "max_content_length": 10000
}
```

## Recommended Workflow

**STANDARD APPROACH (use this most of the time):**
1. Search with `query` and `limit` only - let DuckDuckGo find the best sources
2. Review the 2000-char previews to identify relevant pages
3. Use `html_to_text` on specific URLs for full content

**ADVANCED APPROACH (only when you know what you're doing):**
1. Search with `site` parameter when you're certain of authoritative domains
2. Always include multiple related domains in the array
3. Still use `html_to_text` for full content if needed

**Example Standard Workflow:**
```
Step 1: web_search with basic parameters
{
  "query": "tokio runtime configuration",
  "limit": 5
}

Step 2: Review previews and select best match

Step 3: Read full content
{
  "url": "https://tokio.rs/tokio/topics/runtime",
  "max_content_length": 10000
}
```

**Example Advanced Workflow (when certain about sources):**
```
{
  "query": "async fn trait bounds",
  "limit": 5,
  "site": ["rust-lang.org", "doc.rust-lang.org", "docs.rs"]
}
```

## Key Principles

1. **Default to simple searches** - Use only `query` and `limit` in most cases
2. **Use `site` sparingly** - Only when you know the authoritative domains
3. **Include multiple domains** - Never use `site` with just one domain
4. **Preview first, read second** - Use web_search → html_to_text workflow
5. **Balance context** - Don't request more results or content than you need

Remember: A good search query is often better than complex site filtering. Use `site` only when you are absolutely certain about the authoritative sources.
